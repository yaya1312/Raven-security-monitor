Problem Statement

The Monitoring Gap in Small Security Labs

Modern attacks rarely appear as one obvious event. A single intrusion may start with network probing, continue through port scanning, move into SSH or FTP brute-force attempts, and finish with web path reconnaissance. Raven was built to collect these scattered signals and present them as one clear investigation story.

01

Fragmented Visibility

SSH, FTP, web logs, Nmap findings, and packet events normally live in separate places, making attack chains difficult to see.

02

Manual Log Review

Raw logs are useful, but they are slow to inspect manually. Raven transforms them into structured, queryable security events.

03

Alert Context

Raven links every alert to its evidence, helping analysts understand what happened instead of only seeing a notification.

Architecture

From Raw Log to Unified Dashboard

Raven runs as a single-VM educational proof of concept. Python agents collect logs and packet events, PostgreSQL stores structured evidence, Flask exposes REST endpoints, and React presents the live dashboard.

Feature Showcase

Multi-Protocol Security Visibility

Each module focuses on one part of the environment, while Raven’s correlation layer combines them into a complete security picture.

SSH

SSH Monitoring

Parses authentication successes, failures, disconnects, usernames, source IPs, and brute-force patterns.

FTP

FTP Activity Tracking

Captures login attempts and file operations such as upload, download, delete, mkdir, and rmdir.

WEB

Web Traffic Analysis

Monitors Apache access logs for sensitive paths, error responses, and reconnaissance behaviour.

MAP

Nmap Scan Monitoring

Tracks open services, compares port baselines, and highlights newly exposed ports.

PKT

TShark Packet Capture

Detects ICMP sweeps, DNS bursts, TCP SYN patterns, and HTTP probing at the network layer.

COR

Cross-Source Correlation

Flags IPs active across multiple services and raises critical alerts for multi-stage attack behaviour.

AI

AI-Assisted Insights

Provides analyst-style summaries, risk scoring, correlated IP profiles, and attack-chain interpretation from structured evidence.

API

Flask REST API

Exposes alerts, metrics, agent status, charts, and source-specific event data to the dashboard.

UI

Real-Time Dashboard

Displays live cards, severity rings, log volume charts, alerts, evidence, and agent heartbeat status.

Detection Workflow

Evidence-Driven Security Analysis

Raven follows a clear pipeline: collect, parse, detect, correlate, alert, and investigate.

CollectLogs, packets, scans, and service activity are gathered from the lab VM.
ParseRaw events become structured records with IPs, timestamps, outcomes, and severity.
DetectRule-based logic identifies brute force, probing, scanning, and suspicious activity.
CorrelateRaven connects the same source IP across multiple monitored services.
InvestigateAlerts link to raw evidence and analyst-style summaries for faster review.
AI Insights

AI-Assisted, Not Autonomous

Raven does not claim to use machine learning or self-learning detection. Its detection engine is rule-based. The AI Insights area is framed as an analyst-support workspace that explains risk, summarizes correlated activity, and helps users understand attack chains using database evidence.

Correct positioning: Raven combines rule-based detection and correlation with AI-assisted investigation workflows.
1
Risk ScoreScores IPs using severity, alert volume, failures, web activity, and multi-source correlation.
2
Correlated IP ProfilesShows which IPs appeared across SSH, FTP, web, packet capture, and scan data.
3
Attack Chain ViewOrganizes activity into stages such as reconnaissance, enumeration, access attempts, and exploitation probes.
4
Investigation SummaryTurns structured evidence into readable analyst-style context without claiming black-box ML detection.

Expert Interview

Industry Perspective

The team conducted an expert interview to compare Raven’s design choices with real SOC practices. The feedback highlighted that rule-based detection is still highly relevant for known attack patterns, while AI is most useful as an analyst assistant rather than an automatic decision-maker.

Detection Thresholds

Static thresholds are useful in a lab, but production systems usually require tuning, context, and asset criticality.

Alert Fatigue

Deduplication and cooldown windows help reduce repeated alerts, but mature SOCs also use risk scoring and enrichment.

Scaling Path

To monitor many machines, Raven would need distributed agents, a stronger data pipeline, and scalable storage.

Future Work

A Clear Path Toward a Stronger Platform

Raven is a working educational prototype with a realistic roadmap for future improvement.

Distributed Agents

Extend Raven from one VM to multiple monitored hosts with lightweight remote collectors.

Elastic/OpenSearch

Use scalable search and time-series storage for large log volumes and faster investigations.

Threat Intelligence

Add IP reputation, ASN, geolocation, and public threat-feed enrichment.

Firewall Automation

Introduce optional response actions for high-confidence critical alerts.

MITRE ATT&CK Mapping

Map each detection rule to standard adversary techniques for clearer reporting.

ML Anomaly Detection

Add explainable anomaly models as a complementary layer, not a replacement for rules.
